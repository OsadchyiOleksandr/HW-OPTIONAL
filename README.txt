У цьому завданні вам потрібно буде створити класи для роботи з об'єктами користувачів та їх репозиторієм,
використовуючи клас Optional для забезпечення безпеки та зручності при отриманні об'єктів.

1. Створіть класи для виконання завдань:

a. Створіть клас User, що містить поля id (int), name (String), email (String).

b. Створіть клас UserRepository, який містить список користувачів (List<User>) та методи для роботи з користувачами.

2.Реалізуйте методи:

a. Метод findUserById(int id), який повертає Optional<User> з користувачем за вказаним id.

b. Метод findUserByEmail(String email), який повертає Optional<User> з користувачем за вказаною електронною поштою.

c. Метод findAllUsers(), який повертає список всіх користувачів у вигляді Optional<List<User>>.

3.Створіть методи, щоб продемонструвати різні сценарії використання Optional:

a. Пошук користувача за id. Перевірте, чи знайдений користувач присутній, і виведіть інформацію про нього, якщо він є.

b. Пошук користувача за email. Перевірте, чи знайдений користувач присутній, і виведіть інформацію про нього, якщо він є.

c. Отримання списку всіх користувачів. Перевірте, чи список користувачів присутній, і виведіть кількість користувачів у списку.

4.Залийте виконаний проект в свій GitHub репозиторій, посилання на який зазначте в LMS
